Securing Your Containers- Best Practices for Container Image Management in Azure Kubernetes Service

Sep 17, 2023
2 min read
AKS CI/CD

Containers and container image security are critical priorities when developing and running applications in Azure Kubernetes Service (AKS). Outdated base images or unpatched application runtimes introduce security risks and potential attack vectors. To minimize these risks, you can integrate and run scanning and remediation tools in your containers during both the build and runtime stages. The earlier you identify vulnerabilities or outdated base images, the more secure your application will be.

In this comprehensive guide, we’ll explore best practices for securing your containers in AKS, including:

Secure the Images and Runtime

Best Practice Guidance:

  • Scan your container images for vulnerabilities
  • Only deploy validated images
  • Regularly update the base images and application runtime
  • Redeploy workloads in the AKS cluster

When adopting container-based workloads, it’s essential to verify the security of the images and runtime used to build your applications. To avoid introducing security vulnerabilities into your deployments, follow these best practices:

  • Integrate a process to scan container images using tools like Twistlock or Aqua into your deployment workflow.
  • Only allow verified images to be deployed, for example, by automating image scans, verification, and deployments using a CI/CD pipeline.

Azure Container Registry also includes vulnerability scanning capabilities to help protect your images and registry.

Automatically Build New Images on Base Image Update

Best Practice Guidance:

  • Use automation to build new images when the base image is updated
  • Update any downstream application container images when the base image is updated, as updated base images typically include security fixes

Each time a base image is updated, you should update any downstream container images that use that base image. Integrate this build process into your validation and deployment pipelines, such as Azure Pipelines or Jenkins. These pipelines ensure your applications continue to run on the latest secure base images.

Azure Container Registry Tasks can also automatically update container images when the base image is updated, allowing you to maintain a set of updated base images with bug and security fixes.

For more information about automating base image updates, see the Azure Container Registry Tasks documentation.

By following these best practices for container image management and security, you can significantly reduce the risk of vulnerabilities and attack vectors in your AKS deployments, keeping your applications secure from code to cloud.
